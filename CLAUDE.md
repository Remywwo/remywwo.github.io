# 编码规范

- 按照编程语言最佳实践进行代码编写
- 如果对于用户要求不够明确，可以再次询问确认，严禁自行发挥，擅自修改
- 每个要求的功能实现之后，自动列出改动点以及改动文件
- 未提及让实现具体功能时，严禁对代码进行修改，只回答问题
- 每次要求提交代码时，需要列举具体文件改动点，总结一下，然后让我确认
- 要求创建 PR 时，需要先拉取一下 main 代码到当前分支，看下有没有冲突需要解决，如果有冲突，需要通知用户，严禁自己擅自合并
- 代码PR 必须提供PR链接，严禁擅自合并
- 当我要求统计当前有多少文件改动时，统计内容需要包括：总共有多少文件改动，每个文件的具体改动点，包括改动行数
