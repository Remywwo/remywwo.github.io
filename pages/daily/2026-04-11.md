# 一、主流大模型能力对比

| 维度     | 能力                                                       |

| 推理能力 | ★★★★★ 业界领先，o1/o3 在 MATH、Codeforces 等评测中表现顶尖 |

# 二、大模型能力评估方法

## 1. 主流 Benchmark 介绍

| Benchmark                | 评估维度   | 说明                              |
| ------------------------ | ---------- | --------------------------------- |
| **MMLU**                 | 知识广度   | 57 个学科选择题，考察知识储备     |
| **HumanEval**            | 代码能力   | LeetCode 风格编程题，考察代码生成 |
| **MATH**                 | 数学推理   | 竞赛数学题，考察推理能力          |
| **BBH** (BIG-Bench Hard) | 复杂推理   | 23 道难题，考察多步推理           |
| **IFEval**               | 指令遵循   | 严格按指令执行的能力              |
| **Arena-Hard**           | 人类偏好   | ELO 评分，人类对比投票            |
| **GPQA**                 | 专家级知识 | 研究生水平专业题                  |
| **ARC-C**                | 抽象推理   | 矩阵推理题                        |

## 2. 能力分类评估体系

### A. 基础能力（必测）

- **知识问答**：MMLU、GPQA
- **推理能力**：MATH、BBH、ARC-C
- **代码能力**：HumanEval、MBPP、LiveCodeBench

### B. 进阶能力（核心）

- **指令遵循**：IFEval、BFCL
- **长上下文**：RULER、Needle in Haystack（大海捞针测试）
- **多模态**：MMMU、MathVista、Video-MME

### C. 对齐与安全（必测）

- **Safety**：ToxiGen（毒性检测）、TruthfulQA（真实性）
- **Helpfulness**：MT-bench、Chatbot Arena
- **Bias**：BBQ、CrowS-P

# 三、大模型能力评估流程

## Step 1：明确评估目标

在开始评估前，先明确：

- **场景**：选型、微调评估、上线前验收、定期监控
- **维度**：通用能力、垂直领域、安全合规
- **基线**：是否有参考模型（GPT-4o / Claude 3.7 作为行业标杆）

## Step 2：选择 Benchmark 组合

根据场景选择 3-5 个核心 Benchmark：

```
选型评估：
  - MMLU（知识） + MATH（推理） + HumanEval（代码）
  - MT-bench（综合） + Safety（安全）

上线验收：
  - IFEval（指令遵循） + BBH（推理） + Safety
  - 长上下文测试（RULER / Needle in Haystack）

定期监控：
  - 固定 Benchmark 集 + 业务数据集
  - A/B 测试在线效果
```

## Step 3：准备评测环境

```python
# 标准化 API 调用
def eval_model(model_name, benchmark_data):
    scores = {}
    for test in benchmark_data:
        response = call_api(model_name, test['prompt'])
        scores[test['metric']] = compute_score(response, test['answer'])
    return scores
```

关键点：

- 统一 temperature（通常设 0 或 0.7）
- 统一 max_tokens
- 多次采样取平均（通常 5-10 次）

## Step 4：执行评估

```python
# 典型评估流程
for model in ['gpt-4o', 'claude-3.7', 'gemini-2.5', 'deepseek-v3']:
    result = eval_model(model, BENCHMARK_SUITE)
    save_result(f"{model}_results.json", result)
    print(f"{model}: {result['avg_score']}")
```

## Step 5：结果分析与报告

1. **横向对比**：各模型在各维度的得分雷达图
2. **统计检验**：t-test 或 Bootstrap，验证差异显著性
3. **成本效益**：性能/价格综合评估
4. **场景适配**：根据业务场景加权评分

```
评估报告模板：
## {模型名称} 评估报告

| Benchmark | 得分 | 行业平均 | 差距 |
|-----------|------|----------|------|
| MMLU | 88.2 | 76.5 | +11.7 |
| MATH | 72.1 | 45.0 | +27.1 |
| ... | ... | ... | ... |

结论：[是否满足业务需求，推荐程度]
```

## Step 6：持续监控（上线后）

- 每周跑一次固定 Benchmark
- 监控生产环境的 error rate / latency / user feedback
- 建立回归测试：模型更新前后的 diff

# 四、实践建议

1. **不要只看综合分数**：不同业务场景权重不同，代码场景重点看 HumanEval，教育场景重点看 MATH
2. **Benchmark 疲劳**：主流模型在 MMLU 上都接近饱和（>85），要关注长尾能力
3. **人工评估不可替代**：自动评测无法覆盖创意写作、对话风格等主观维度
4. **成本是真实约束**：DeepSeek V3 性能接近 GPT-4o 但价格低 50 倍，合理权衡
